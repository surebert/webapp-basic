Beginning A Surebert Framework

If you have questions you can email paul.visco AT roswellpark.org

  1. To begin, make sure that you have a copy of the surebert framework deployed to your server. Ani, obi, billy, and the fedora vmware image already have it installed in /var/www/sbf You can find up to date deployment intructions on the surebert framework quickstart guide.

  2. Next, set your SUREBERT_FRAMEWORK_SB_PATH in /private/config/definitions.php so that it matches the path that the sbf is deployed at. e.g. 
    define("SUREBERT_FRAMEWORK_SB_PATH", "/var/www/sbf/sb/1.0");
  3. Next, set your SUREBERT_TOOLKIT_PATH in /private/config/definitions.php so that it matches the path that the sbf is deployed at. e.g. 
    define("SUREBERT_TOOLKIT_PATH", "/var/www/sbf/toolkit/4.64");
  4. OPTIONAL: If using Roswell Park rp modules, set your SUREBERT_FRAMEWORK_RP_PATH in /private/config/definitions.php so that it matches the path that the sbf is deployed at. e.g. 
    define("SUREBERT_FRAMEWORK_RP_PATH", "/var/www/sbf/rp/1.0");
  5. Finally, In order to get rid of this page, you need to go ahead and edit /public/.htaccess and comment change 
    RewriteRule .* index.html
    to 
    #RewriteRule .* index.html
    TIP: You can use this index.html as a static placeholder in an emergency, say if your DB is down. You may want to keep it around and remember what to uncomment to turn it back on!
  6. Visit your site, you should be ready to go! Read up on the surebert framework and toolkit @ surebert.com and on the surebert wiki
